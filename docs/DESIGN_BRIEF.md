# OpenRAG-Lab — Frontend Design Brief

> 디자인 툴(예: Claude Artifacts, v0, Lovable, Figma Make)에 그대로 붙여넣어
> 4개 화면 시안을 만들 수 있도록 작성된 단일 프롬프트.
> 출처: REQUIREMENTS_v4 / ARCHITECTURE / API_SPEC_v4 / 현재 구현 상태(2026-04-29).

---

## 1. 한 줄 정의

**OpenRAG-Lab**은 사용자가 자기 PC의 개인 문서로 RAG 시스템을 만들고,
임베딩·청킹·검색·LLM 조합의 품질을 정량적으로 비교·튜닝하는 **로컬 데스크톱 워크벤치**다.
모든 데이터는 로컬에 머물고, 외부 LLM API는 사용자가 명시적으로 키를 등록한 경우에만 호출된다.

데이터 도구다 — Linear / Vercel Dashboard / Datadog / Posthog 류의 정보 밀도와 차분함에,
**샤넬(Chanel) 모노크롬의 절제된 럭셔리**를 입힌다. 블랙·화이트·크림·골드의 4색 세계.
"슬릭한 SaaS 랜딩"이 아니라 "내가 매일 쓰는 작업대"의 분위기 — 단, 그 작업대는 흑단과 진주의 톤.

## 2. 핵심 가치 (디자인이 시각적으로 표현해야 할 것)

| 가치 | 디자인이 보여줘야 하는 것 |
|---|---|
| **프라이버시** | 모든 처리가 로컬임을 항상 인지 가능. 외부 호출은 명시적 시각 신호. |
| **투명성** | 청킹 결과·검색 컨텍스트·평가 점수까지 RAG 파이프라인 내부가 다 보인다. 답변에는 항상 인용/출처. |
| **반복 실험성** | 같은 데이터로 여러 설정을 돌리고 결과를 나란히 비교할 수 있다. fingerprint(sha256[:16])가 실험의 ID. |
| **3 OS 동등** | macOS·Windows·Linux를 동등하게 다룸. 특정 OS 메타포(ex: macOS 트래픽 라이트) 금지. |

## 3. 사용자 페르소나 → 화면 매핑

| 페르소나 | 한 줄 | 주로 쓰는 화면 |
|---|---|---|
| **지수** (변호사, 비전문가) | 폴더 드래그→ 한 클릭→ 채팅. 5분 안에 도달. | Auto-Pilot, Chat |
| **민호** (RAG 학습자) | 청크 크기·전략을 슬라이더로 바꿔 시각적으로 체감. | Chunking Lab, Chat |
| **수진** (사내 RAG 엔지니어) | 임베더×검색×LLM 조합 매트릭스 일괄 평가. | Experiment Matrix, Chat |

비전문가와 엔지니어가 같은 앱을 쓴다. **고급 기능은 숨겨두지 말고 옆에 두되, 기본 흐름은 비전문가 기준.**

## 4. 정보 아키텍처

전역 헤더 + 4개 라우트의 SPA. 모달은 라이선스 수락·임베더 차원 변경 동의 등 게이팅 시에만.

```
/                — Auto-Pilot Wizard       (지수의 "그냥 시작")
/chunking        — Chunking Lab            (민호의 "체감")
/chat            — Chat                    (모두의 공용 검색·답변)
/experiments     — Experiment Matrix       (수진의 "비교 보고")
```

워크스페이스는 동시에 여러 개 보유 가능. **활성 워크스페이스**는 헤더에서 항상 보이고 전환 가능하다.

## 5. 글로벌 UI

### 5.1. 헤더 (모든 화면 공통, 상단 고정)

좌측부터:
- **로고 / 앱 이름**: `OpenRAG-Lab` — 작고 정적, 강조하지 않음.
- **워크스페이스 셀렉터**: 드롭다운. 현재 활성 워크스페이스명 + 문서/청크 수 작게.
- **네비게이션**: Auto-Pilot · Chunking Lab · Chat · Experiments (현재 라우트 underline).

우측부터:
- **시스템 상태 칩** (작게):
  - OS · CPU · RAM · GPU acceleration backend (`cuda` / `metal` / `cpu`) 표시.
  - 클릭 시 하드웨어 프로필 전체 팝오버.
- **네트워크 인디케이터**: 기본 회색 점("local only"). 외부 LLM 호출 중이면 호박색 점 + 제공자명("Anthropic"). 호출 끝나면 자동으로 회색.
- **테스트 모드 배지** (`OPENRAG_LAB_TEST_MODE=1`일 때만): 작은 라벨 "Test mode — fake adapters".

### 5.2. 빈 상태·로딩·에러 — 각 화면이 반드시 가져야 함

- **Empty state**: 일러스트보다 한 줄 안내 + 다음 행동(버튼 1개). 과한 그래픽 금지.
- **Loading**: 스켈레톤 또는 프로그레스 바. 스피너는 짧은 작업에만.
- **Error envelope**: API가 항상 `{code, message, recoverable, details}`를 반환한다. 토스트 + 인라인 메시지. `recoverable=true`면 재시도 버튼, `false`면 무엇이 깨졌는지·문서 링크.

### 5.3. 검색 전용 모드(retrieval-only) 배지

`llm_id`가 비어 있는 워크스페이스/실험에서 항상 표시되는 라벨. Chat 헤더, Experiment 행, Auto-Pilot 결과에 모두 등장.
디자인: 외곽선 칩 + "Retrieval-only" 텍스트. **부정적 색이 아니라** "이런 모드도 있다"는 정보 라벨.

### 5.4. 외부 LLM 사용 표시

외부 LLM이 호출되는 모든 단계는 UI에 명시한다 (제공자, 어떤 단계에서 호출). 비밀 정보·API 키는 절대 노출 금지.

## 6. 화면별 상세

### 6.1. `/` — Auto-Pilot Wizard

**목적**: 비전문가가 5분 안에 채팅까지 도달. 선택지를 최소화.

3단계 수직 흐름. 각 단계는 명확히 분리된 카드. 이전 단계가 완료돼야 다음이 활성화.

#### Step 1 — 프리셋 선택
- 진단 결과 카드(상단): "Apple M2 Pro · 32GB RAM · Metal" 형태.
- 프리셋 3종 카드 가로 배치:
  - **속도 우선** (Lite) — 경량 임베더, 작은 청크.
  - **밸런스** (Balanced) — 중간 모델, 표준 청크. ★ 추천 배지.
  - **정확도 우선** (Quality) — 큰 모델, 시맨틱 청킹.
- 각 카드: 이름 / 한 줄 사유 / 모델 메타데이터 / "이 PC에서 가용" 또는 "RAM 부족" 회색 처리.
- 자동 추천된 카드는 미리 선택돼 있다.

#### Step 2 — 워크스페이스 + 문서
- 워크스페이스 이름 입력.
- **드래그-앤-드롭 영역**: 큰 점선 박스, "PDF · TXT · Markdown 파일을 여기로 끌어다 놓으세요". 파일 단위와 폴더 단위 모두 가능.
- 업로드된 파일 목록: 파일명 / 크기 / 형식 아이콘 / 제거 버튼.
- 변환 실패 파일은 **별도 리스트**로 표시 (전체 작업은 계속 진행).

#### Step 3 — 시작 + 진행률
- "시작" 버튼 (primary). 누르면 워크스페이스 생성 → 문서 업로드 → 인덱싱 시작.
- 인덱싱 진행: 단계별 진행률 (parsed / chunked / embedded). 문서 단위로도, 전체로도 보여줌. WebSocket 실시간.
- **체크포인트 표시**: 중단/재개 가능함을 작은 텍스트로 안내.
- 완료 시 "→ 채팅으로 가기" 버튼.

### 6.2. `/chunking` — Chunking Lab

**목적**: 청크 분할 전략이 검색 품질에 미치는 영향을 시각적으로 체감.

좌·우 2-패널 레이아웃 (1024px 기준 50:50, 그 이하는 위/아래).

#### 좌측 — 컨트롤
- 워크스페이스의 문서 중 1개 선택 (드롭다운).
- 청킹 전략 라디오:
  - Fixed (P0)
  - Recursive (P0, 한국어 경계 포함)
  - Sentence (P1, 비활성)
  - Semantic (P1, 비활성)
- chunk_size 슬라이더 (예: 128 ~ 2048).
- overlap 슬라이더 (예: 0 ~ 512).
- 실시간 메트릭: **청크 개수 / 평균 토큰 / 최소 / 최대**. 슬라이더 움직일 때 200ms debounce 후 갱신.

#### 우측 — 미리보기
- 원본 문서 텍스트 위에 **청크 경계가 색상으로 오버레이**. 인접 청크끼리 다른 색.
- overlap 영역은 두 색이 겹치는 빗금/반투명.
- 페이지 메타데이터(있다면)와 청크 ID 호버 표시.
- 1초 이내 미리보기 반영이 핵심 UX 요건.

상단에는 "이 설정으로 인덱싱 실행" CTA — 누르면 새 실험으로 진행.

### 6.3. `/chat` — Chat

**목적**: 답변과 그 근거를 한 화면에서 추적.

3-컬럼 레이아웃:

#### 좌측 (좁음, 240px) — 실험 리스트
- 이 워크스페이스의 실험들. 각 항목:
  - fingerprint 앞 6자 / chunking + embedder 요약 / 상태(completed/archived).
  - **archived** 실험(이전 임베더 차원)은 흐리게 + "archived" 라벨.

#### 가운데 (가장 넓음) — 대화
- 메시지 버블이 아니라 **트랜스크립트 스타일** (Q/A 분리, 좌측 정렬).
- 답변 안의 인용 마커: `[1]`, `[2]` — 클릭하면 우측 패널의 해당 청크로 스크롤·하이라이트.
- 답변 상단에 메타: latency / 사용된 청크 수 / 모드 배지(retrieval-only면 답변 패널 자체가 안 보이고 "검색된 청크" 메시지만).
- 외부 LLM이 답변을 생성한 경우, 답변 하단에 작은 "via OpenAI gpt-4" 라벨.

#### 우측 (320px) — 컨텍스트 패널
- 검색된 청크 카드들. 각 카드:
  - 출처 파일명 + 페이지 (예: `case_2024.pdf · p.42`).
  - 유사도 점수 (0~1, 우측 상단).
  - 청크 본문 (자르지 말고 스크롤).
  - 인용된 청크는 좌측에 컬러 바.

### 6.4. `/experiments` — Experiment Matrix

**목적**: 수진(엔지니어)의 비교·보고서 작성 흐름.

#### 상단 — 매트릭스 정의 (P1 영역, 자리 잡아두기)
- 임베더 × 검색 전략 × LLM 후보를 컬럼/행으로 정의하는 셀 그리드. (P0 시안에서는 placeholder만, 비활성 처리.)

#### 가운데 — 평가 결과 표 (P0 핵심)
- 행: 각 실험 (fingerprint + 설정 요약).
- 열: **RAGAS 4지표** — Faithfulness · Answer Relevance · Context Precision · Context Recall.
- 셀: 0.00~1.00 숫자 + 색상 (녹→황→적 그라데이션).
- 검색 전용 실험은 LLM 의존 지표(Faithfulness, Answer Relevance) 셀이 `—` (회색)이고 호버 시 "retrieval-only mode" 툴팁.
- 헤더 클릭으로 정렬, 행 선택으로 A/B.

#### 하단 — A/B 비교 차트
- 두 실험을 선택하면 그룹 막대그래프(Recharts).
- 자동 생성 요약 라벨: "설정 A가 Faithfulness +0.12 우세, Context Recall -0.05 열세".
- "YAML로 익스포트" / "골든 셋 보기" 버튼.

#### 골든 셋 관리 (서브 패널 또는 모달)
- 질문-기대답변 쌍 CRUD. CSV 업로드. 최소 10개 이상 권장 안내.

## 7. 모달·게이팅

다음 시점에는 명시적 모달.
- **임베더 차원 변경 감지**: "기존 인덱스가 무효화되며 전체 재인덱싱이 필요합니다 (예상 X분). 이전 실험은 archived로 보존됩니다." → 동의 후 진행.
- **외부 LLM 활성화**: 키 미등록이면 등록 폼 모달. 자동 진행 절대 금지.
- **모델 라이선스 수락** (P1): 사용자 수락 요구 라이선스(Llama 3 등)는 본문 + "동의" 클릭.

## 8. 디자인 토큰

### 8.1. 컬러 — Chanel Monochrome

샤넬의 시그니처는 **블랙 · 화이트 · 크림(펄) · 골드**의 4색. 채도 있는 색은 거의 쓰지 않는다.
정보 도구의 가독성을 위해 그레이스케일 단계를 잘게 두고, 의미 신호(상태)도 **데사처레이트된 톤**으로 절제한다.

#### 베이스 — 듀얼 테마
| 토큰 | Light (Pearl) | Dark (Noir) | 용도 |
|---|---|---|---|
| `bg/0` | `#FAF7F2` (cream/pearl) | `#0A0A0A` (true black) | 페이지 배경 |
| `bg/1` | `#F2EEE6` | `#141414` | 패널·헤더 |
| `bg/2` | `#E8E2D6` | `#1C1C1C` | 카드 |
| `border` | `#1A1A1A @ 8%` | `#FFFFFF @ 10%` | 1px hairline |
| `text/0` | `#0A0A0A` | `#F5F1E8` (off-white) | 본문 |
| `text/1` | `#3A3A3A` | `#B8B2A4` | 보조 |
| `text/2` | `#7A7468` | `#6E6A60` | 메타·캡션 |

다크가 시그니처(샤넬 No.5의 검정 캡 위 금문자) — **다크 1급**, 라이트는 세컨더리.

#### 악센트 — Champagne Gold (단 한 가지)
- `accent` = `#C8A96A` (샹파뉴 골드). hover `#D4B97C`, pressed `#B89856`.
- 중요한 CTA·활성 라우트 underline·차트 1차 시리즈·인용 마커 `[1]`·선택된 셀.
- **남용 금지**: 한 화면에 골드는 1~2 군데. 나머지는 모두 그레이스케일.

#### 상태 — 절제된 톤 (데사처레이트)
| 의미 | Hex | 비고 |
|---|---|---|
| Success | `#7A8B6F` (sage) | 인덱싱 완료, 점수 양호 |
| Warn | `#C8A96A` (gold = accent와 공유) | 외부 LLM 호출 중, 임베더 차원 변경 경고 |
| Error | `#8B3A3A` (burgundy/wine) | 실패. 빨강이 아니라 와인 톤 |
| Info | `#6E6A60` (warm gray) | 일반 알림 |

샤넬 무드 유지를 위해 형광 채도 금지. 토스트·뱃지 채움보다 **외곽선 + 텍스트 컬러**로 신호 전달.

#### 차트 — Monochrome + Gold
- 5-시리즈 정성적 팔레트: `#0A0A0A` · `#C8A96A` · `#7A7468` · `#B8B2A4` · `#3A3A3A`.
- 라이트 모드에서는 명도 반전.
- 색맹 안전성은 **명도 차 + 패턴**(채움 / 외곽선 / 점선 / 빗금)으로 확보.
- A/B 비교 막대그래프: 한 시리즈는 골드, 다른 하나는 차콜 — 직관적이면서도 톤이 깨지지 않음.

#### 청킹 오버레이
- 6색 순환 대신 **그레이스케일 6단계 + 골드 1**로 변경: `#0A0A0A` · `#3A3A3A` · `#6E6A60` · `#9A9488` · `#B8B2A4` · `#C8A96A`. 모두 35% 투명도.
- 인접 청크끼리 명도 차로 구분. overlap 영역은 두 톤이 겹쳐 자연스러운 빗금감.

#### RAGAS 점수 셀 그라데이션
0.00→1.00을 빨강→노랑→초록 대신 **차콜→웜그레이→골드**로 매핑. 높을수록 골드.

### 8.2. 타이포

샤넬의 타이포는 **얇은 산세리프 + 넓은 자간**(Coco / Vogue 스타일). 데이터 도구 가독성과 균형을 맞춘다.

- **헤딩**: Inter / Pretendard의 **300 (Light) 또는 400 (Regular)**. 굵은 weight 금지.
  - 큰 헤딩(28px)은 letter-spacing `0.02em`로 살짝 트래킹.
  - 모두 대문자(`text-transform: uppercase`)는 §6.1 Step 라벨, 섹션 캡션 등 짧은 라벨에만.
- **본문**: Inter 또는 Pretendard 400. 한글-영문 혼용 시 자간 일관성 위해 동일 패밀리 권장.
- **숫자/메트릭**: tabular-nums. 점수·진행률·fingerprint는 mono(JetBrains Mono · IBM Plex Mono) 권장.
- **시그니처 헤더 옵션**: 앱 로고 `OpenRAG-Lab` 워드마크는 Didot/Bodoni 류 **하이콘트라스트 세리프**도 가능 (샤넬 워드마크 무드). 본문엔 절대 사용 금지.
- 크기: 12/13/14/16/20/28 6단계.

### 8.3. 레이아웃
- 데스크톱 우선. 1280 / 1440 / 1920 세 폭에서 자연스럽게.
- 모바일 디자인 **하지 않음** (Non-goal).
- 그리드 8px 베이스. 카드 모서리 8px, 큰 카드 12px.
- 그림자 거의 없음 (데이터 툴 분위기). 경계는 1px border + 미세한 표면 명도 차.

### 8.4. 인터랙션
- 호버는 명도 미세 변경, 클릭 가능 영역은 cursor pointer.
- 슬라이더·차트 등 데이터 도구는 키보드 접근성 필수.
- 토스트 우측 하단 4초.
- 페이지 전환 fade 80ms 미만.

## 10. 산출물 요청 (디자인 툴에 줄 task)

다음 4개 화면의 **데스크톱 시안** (1440px 기준):

1. **Auto-Pilot — 진행 중 상태**: 프리셋 선택됨 + 파일 5개 업로드됨 + 인덱싱 60% 진행 중.
2. **Chunking Lab**: Recursive · chunk_size=512 · overlap=64로 한국어 PDF 1개 미리보기. 청크 18개 색상 오버레이.
3. **Chat — 답변 + 인용**: 질문 1개 답변 1개. 우측 컨텍스트에 청크 4개. `[1]`·`[2]` 인용 클릭 가능.
   - 추가 변형: **retrieval-only 모드**(답변 패널이 "검색된 청크 보기" 안내로 대체).
4. **Experiment Matrix**: 4개 실험(Lite/Balanced/Quality + retrieval-only 1개) RAGAS 4지표 표 + 하단 A/B 막대그래프.

각 화면에 대해:
- 기본 상태 + 1~2개 주요 변형(에러·empty·다크 모드).
- 컴포넌트 인벤토리(버튼·인풋·카드·칩·모달) 1매.
- 컬러·타이포 토큰표 1매.
